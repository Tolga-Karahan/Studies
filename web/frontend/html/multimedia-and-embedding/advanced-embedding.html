Advanced embedding

iframes are for embedding other web pages, embed and object are for embedding external resources such as PDF.

iframes

<iframe> elements are designed to embed other web documents into the current document. It is very useful to
incorporate third party content instead of coding them by ourselves such as video from video providers,
commenting sytems, maps, advertising banners etc.

Security concerns for iframe

iframes become target for hackers to maliciously modify the site, trick people to do harmful things etc, so
engineers and developers added various security mechanisms. One such iframe attack is clickjacking where
hackers embed an invisible iframe into victim's document or embedding their document into their document
to capture user interactions and sensitive information. Some useful security precautions are:

  • HTTPS: Websites should be served using HTTPS whenever possible. It prevents remote content to be
    examined and prevents embedded content from accessing content in the parent document or vice versa.
  • sandbox attribute is always be used to give only the permissions needed for the external content to do
    its job.
  • Configuring CSP directives can prevent malicious behaviour. CSP stands for content security policy and
    provided a set of HTTPS headers to improve security of the HTML document. For example to prevent
    other sites from embedding our own content we can send and appropriate X-Frame-Options header.

<embed> and <object> elements
