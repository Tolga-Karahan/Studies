<col> and <colgroup> elements

<col> and <colgroup> elements are used to style an entire column at once. After opening table
tag, we can add a colgroup element in which contains col elements for each column to style each
column. Also, we can use span attribute of col element to apply same styling to multiple columns.

Adding captions

Captions can be added to table via <caption> element to give a description for the table which is
useful for users and people with visual impairments. Instead of reading or listening table content
from screen reader, users can have an idea regarding table's data about what.

Adding structure

As tables get more complex, it is useful to add more structure to them by using <thead>
<tfoot> and <tbody> elements. These elements doesn't improve table visually nor provide
more accessibility for screen readers, but they are useful for styling and layout. For example
if table is too long then table header and footer can be repeated on every page and table
body can be displayed on a single page also by adding scrolling up and down.

Accessibility

We make associations between rows and columns of a table to understand underlying
data, but it is not as easy for people who have visual impairments as people who don't.
Screen readers identify all headers and use them to make programmatic associations
between headers and the cells they relate to. There also some useful elements and
attributes for accessibility.

scope attribute
scope attribute in <th> element tells screen reader for which cells the header
is header for. Is it a header for a row or a column? It allows screen readers to read
entire column or row at once. For example if the header is a header for the
column: <th scope="col">header1</th> If header spans multiple rows or columns
then rowgroup or colgroup values must be used.
Alternative way to scope is using id and header attributes.
Each header define a unique id and each <td> element defines their headers in
headers attribute.
