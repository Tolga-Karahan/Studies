Basics of HTML Document Structure

There are several HTML block elements to define areas of websites such as header, navigation menu, main content column.

Basic sections of a document
header
Usually a big strip across the top with a big heading, logo and a possible tagline.
<header> tag is the intended HTML tag for headers.
navigation bar
Contains links to the sections of the site. Usually includes menu buttons, links, or tabs.
<nav> tag is the intended HTML tag for navigation bars.
main content
Big area in the center that is usually unique for each page.
<main> is the intended HTML tag for main content. Content subsections are represented by <article>, <section>
and <div> elements.
article
<article> element encloses a block of related content that makes sense by its own without the rest of the page.
section
<section> is similar to <article> but it is more intended for grouping together a part of the page that constitutes
one single functionality or a theme. It is best practice to begin each sections with a heading. Depending on the
context articles can be split into different sections or sections can be split into different articles.
sidebar
Provides additional indirect information regarding main content and sometimes behaves like a seconday navigation
system. For example if the main content is news then sidebar may include info related to author.
<aside> is intended HTML tag for side bars. It often placed inside <main>.
footer
A strip across the bottom of the page. Usually contains fine print, copyright, or contact info.
<footer> is intended HTML tag for footers.
If we want to group some elements but can't find a proper semantic wrapper or if we don't want to add any specific meaning then we can use <div> and <span> elements.
span
<span> is an inline non-semantic element.
div
<div> is a block level non-semantic element.

Design steps for a website

  1. Some content such as navbar and footer will be most likely the same throughout pages, so note down what you want to have common to every page.
  2. Draw a sketch of each page's structure.
  3. Brainstorm all the content that is not common to every page.
  4. Group all the content that might live together on different pages.
  5. Sketch a sitemap. Have a bubble for each page and draw lines to show the workflow.
